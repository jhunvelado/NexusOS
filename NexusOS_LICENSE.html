Deployment Guide
Host it yourself in 15 minutes — you control access
✅You share a URL, never a file. Access is controlled through Supabase Auth — revoke a user instantly by disabling their account in the Supabase dashboard.
1
Create a free GitHub account Go to github.com → Sign up using your personal email (not a work email). This is critical for IP ownership separation.
2
Create a repository named "NexusOS" Click "New repository" → name it "NexusOS" → set to Public (required for GitHub Pages free hosting) → click Create.
3
Upload index.html and nexusos-app.html Click "Add file" → "Upload files" → drag both HTML files → commit directly to main.
4
Enable GitHub Pages Go to Settings → Pages → Source: "Deploy from branch" → Branch: main → Save. Your app goes live at yourusername.github.io/NexusOS/ within 2 minutes.
5
Connect your Supabase project Create a free project at supabase.com. The app's config block at the top of nexusos-app.html already holds your Supabase URL and publishable key. Run the SQL migration from the comment block at the top of the file to set up all tables and RLS policies.
// Already in nexusos-app.html — verify these match your project: const SUPABASE_URL = 'https://your-project.supabase.co'; const SUPABASE_KEY = 'sb_publishable_...'; // public key only
6
Create user accounts in Supabase Auth Go to Supabase Dashboard → Authentication → Users → "Invite user." Each client gets their own email/password login. Access is workspace-isolated via Row Level Security — no client can see another's data.
7
Revoke access instantly when the engagement ends Go to Supabase → Authentication → Users → find the client → click "Ban user" or delete their account. They are locked out immediately — no file changes, no redeployment needed.
// Or use the Supabase Admin API from an Edge Function: await supabaseAdmin.auth.admin.deleteUser(userId); // Client loses all access within seconds.
8
Set their subscription plan in the workspaces table In Supabase → Table Editor → workspaces → find their row → set plan to starter, professional, or enterprise. Plan-gated features lock or unlock instantly on next page load.
9
What to say when you share access with the client "Here are your NexusOS credentials. Access is via the secure hosted link below — I manage the platform and your workspace is fully isolated from any other client. Your login is personal to you and remains active while we're working together."
